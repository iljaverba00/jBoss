Клиенты Geocad Enterprise Edition

На этой странице находится информация по автоматически обновляемым клиентам Geocad Enterprise Edition, а так же инсталлятор, конфигурирующий среду для их установки и запуска.

Общие сведения

Автообновляемый клиент GEE - это обычное с точки зрения функционала клиентское приложение, которое благодаря использованию технологий Java Web Start и Java Network Launching Protocol может автоматически обновлять свой исполняемый код с сервера приложений. При первом запуске приложения его код скачивается на компьютер пользователя, при последующих запусках выполняется проверка на наличие обновлений, при необходимости они так же скачиваются и без дополнительного участия пользователя устанавливаются.

Такая технология позволяет пользователю быть уверенным, что он работает с самой последней версией приложения, доступной для данного сервера и соответствующей его функционалу.

Инсталлятор клиентов

Для простой установки клиентов GEE в операционной среде MS Windows рекомендуем воспользоваться специально разработанным инсталлятором. Он создает в указанной папке несколько небольших исполнимых файлов, а так же ярлыки для них в стандартном меню Пуск. Установка выполняется для текущего пользователя, соответственно, и выполнять ее нужно в сеансе пользователя, который далее будет пользоваться приложениями.

В процессе установки будет предложено ввести адрес (URL и порт) сервера, откуда будут загружаться и обновляться клиент-приложения. Обычно это тот же сервер, на котором происходит работа с данными. Может быть установлено несколько клиентов для разных серверов; в таком случае допускается установка в одну папку на диске и в одно меню кнопки Пуск.

Взять инсталлятор можно по следующей ссылке: gee_clients.exe (примерно 100 кБ). Предварительно требуется установка Java Runtime Environment 5.0 (JRE) и Java Advanced Imageing (JAI). Скачать их можно по следующим адресам:
JRE 5.0, версия не ниже 5.0 Update 14 (в данный момент по ряду причин JRE версии 6 и выше не поддерживаются);
JAI, версия для JRE не ниже 1.1.3.

Технические подробности

Инсталлятор создает в указанной папке для каждого выбранного клиент-приложения пакетный (batch) файл. Например, для GEE InfoForms Pro это будет infoforms URL.bat, где URL - адрес сервера, введенный при установке. Содержимое этого файла примерно таково:

SET JAVAWS_VM_ARGS=-Xmx64m -Dlogfile=infoforms.log
javaws http://url/wc_client/infoforms.jnlp

В первой строчке задаются параметры запуска Java-машины, в данном примере - количество выделяемой памяти (64 мегабайта) и имя файла отладочного протокола работы. Во второй строчке происходит запуск Java Web Start для указанного URL приложения. В принципе, администратор может сам создать такой файл и настроить запуск, без использования инсталлятора.

Помимо файлов запуска инсталлятор создает пакетный файл remcache URL.bat, который в момент удаления клиентов чистит кэш Java Web Start. В нем для каждого приложения прописана строка вида:

javaws -uninstall http://url/wc_client/infoforms.jnlp
